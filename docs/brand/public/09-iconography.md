## 2. CONSTRUCTION GRID

### 2.1 Base Grid

All icons built on a 24×24 pixel grid:

```
┌────────────────────────┐
│  ┌──────────────────┐  │
│  │                  │  │
│  │    Safe Zone     │  │
│  │      20×20       │  │
│  │                  │  │
│  └──────────────────┘  │
│      2px padding       │
└────────────────────────┘
```

**Grid Properties:**
- Outer boundary: 24×24px
- Safe zone: 20×20px (2px padding)
- Optical center: 12×12
- Keyline guides at 2px intervals

### 2.2 Keyline Shapes

Standard keyline templates:

**Square:**
- 20×20px
- Centered in grid
- For blocky icons

**Circle:**
- 20px diameter
- Centered in grid
- For circular icons

**Vertical Rectangle:**
- 16×20px
- Centered horizontally
- For tall icons

**Horizontal Rectangle:**
- 20×16px
- Centered vertically
- For wide icons

### 2.3 Stroke Standards

**Primary Stroke:**
- Width: 2px
- Cap: Round
- Join: Round
- Consistent throughout icon

**Detail Stroke:**
- Width: 1.5px (when needed)
- Used for internal details
- Maintains visual weight

## 3. STYLE SPECIFICATIONS

### 3.1 Outline Style (Primary)

The default icon style:

**Characteristics:**
- 2px stroke, no fill
- Round line caps and joins
- 2px corner radius minimum
- Single continuous path preferred

**Application:**
- Navigation icons
- Action icons
- Status icons
- Most UI contexts

### 3.2 Filled Style (Active)

For selected/active states:

**Characteristics:**
- Solid fill
- Same silhouette as outline
- May include internal negative space
- Maintains recognizability

**Application:**
- Active navigation items
- Selected states
- Emphasis moments

### 3.3 Glow Style (Emphasis)

For special emphasis:

**Characteristics:**
- Outline base with glow effect
- Color matches context
- Subtle animation optional
- Reserved for key moments

**Application:**
- Rewards
- Achievements
- Special features
- Entity-related content

## 5. SIZING SYSTEM

### 5.1 Size Scale

| Name | Size | Usage |
|------|------|-------|
| XS | 16px | Inline, badges |
| SM | 20px | Dense UI, lists |
| MD | 24px | Standard UI (default) |
| LG | 32px | Emphasized actions |
| XL | 48px | Feature icons |
| 2XL | 64px | Hero, empty states |

### 5.2 Size Context

**16px (XS):**
- Badge accompaniments
- Inline text icons
- Very dense interfaces
- Stroke: 1.5px

**20px (SM):**
- List item icons
- Secondary actions
- Compact navigation
- Stroke: 1.5px

**24px (MD):**
- Standard interface icons
- Primary navigation
- Action buttons
- Stroke: 2px

**32px (LG):**
- Featured actions
- Modal actions
- Emphasized content
- Stroke: 2px

**48px (XL):**
- Feature introductions
- Empty state graphics
- Key actions
- Stroke: 2.5px

**64px (2XL):**
- Hero moments
- Onboarding
- Major states
- Stroke: 3px

### 5.3 Touch Targets

Regardless of icon size:
- Minimum touch target: 44×44px
- Centered within target area
- Adequate spacing from neighbors

## 7. ICON ANIMATION

### 7.1 State Transitions

**Color Change:**
```css
.icon {
  transition: color 150ms ease-out;
}
```

**Scale on Press:**
```css
.icon:active {
  transform: scale(0.92);
  transition: transform 50ms ease-out;
}
```

**Glow on Hover:**
```css
.icon:hover {
  filter: drop-shadow(0 0 4px currentColor);
  transition: filter 200ms ease-out;
}
```

### 7.2 Loading Animation

**Spinner:**
```css
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

.icon-spinner {
  animation: spin 1s linear infinite;
}
```

### 7.3 Notification Pulse

**Badge Pulse:**
```css
@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.2); }
}

.icon-badge {
  animation: pulse 2s ease-in-out infinite;
}
```

## 8. ICON PAIRING

### 8.1 Icon + Text

**Horizontal Pairing:**
- Icon left, text right
- 8px gap between
- Vertical center alignment

**Vertical Pairing:**
- Icon above, text below
- 4px gap between
- Horizontal center alignment

### 8.2 Icon + Badge

**Position:**
- Badge at top-right of icon
- Overlapping icon boundary
- Minimum 8px badge size

**Style:**
- Solid background
- High contrast text
- Circular or pill shape

### 8.3 Icon Groups

**Spacing:**
- Minimum 16px between icons
- Consistent spacing throughout
- Group alignment maintained

## 9. ICON ACCESSIBILITY

### 9.1 Text Alternatives

All icons must have:

**aria-label:**
```html
<button aria-label="Close dialog">
  <svg><!-- close icon --></svg>
</button>
```

**Screen Reader Text:**
```html
<span class="sr-only">Close dialog</span>
```

### 9.2 Color Independence

Icons should be recognizable without color:
- Shape conveys meaning
- Context provides clarity
- Labels accompany when needed

### 9.3 Motion Sensitivity

Animated icons should:
- Respect reduced-motion preference
- Have static alternative
- Not be sole state indicator

## 10. ICON CREATION GUIDELINES

### 10.1 New Icon Process

1. **Research**
   - Check existing icons
   - Research metaphors
   - Consider context

2. **Sketch**
   - Multiple concepts
   - Grid adherence
   - Simplicity focus

3. **Build**
   - 24×24 grid
   - 2px stroke
   - Round joins/caps
   - Clean paths

4. **Test**
   - All sizes
   - All states
   - Context placement
   - Accessibility

5. **Export**
   - SVG optimized
   - Consistent naming
   - Documentation

### 10.2 Quality Checklist

- [ ] Fits 24×24 grid
- [ ] 2px stroke weight
- [ ] Round caps and joins
- [ ] Within safe zone
- [ ] Optically balanced
- [ ] Clear at 16px
- [ ] Meaningful metaphor
- [ ] Consistent with set
- [ ] All states designed
- [ ] SVG optimized

## 12. SVG OPTIMIZATION

### 12.1 Clean SVG Structure

```xml
<svg width="24" height="24" viewBox="0 0 24 24" 
     fill="none" xmlns="http://www.w3.org/2000/svg">
  <path d="..." 
        stroke="currentColor" 
        stroke-width="2" 
        stroke-linecap="round" 
        stroke-linejoin="round"/>
</svg>
```

### 12.2 Optimization Rules

- Remove unnecessary attributes
- Use `currentColor` for stroke
- Minimize path complexity
- No embedded styles
- No unnecessary groups
- Clean decimal precision (2 places)
