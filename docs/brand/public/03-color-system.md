## 1. COLOR PHILOSOPHY

### 1.1 The Role of Color

In the M1SSION™ visual system, color serves as a critical communication tool. Unlike brands that use color primarily for aesthetic purposes, M1SSION™ employs color as a functional language that conveys meaning, state, and urgency.

**Color Principles:**

1. **Meaning Over Decoration**
   - Every color choice communicates information
   - Users learn to read color as language
   - Consistency is paramount for comprehension

2. **Restraint Creates Impact**
   - Dark base allows colors to command attention
   - Limited palette increases each color's significance
   - Color is earned, not freely applied

3. **Glow as Personality**
   - Luminous colors are the brand signature
   - Glow effects extend color influence
   - Light seems to emit from within

4. **Contrast for Accessibility**
   - All color combinations meet WCAG AA minimum
   - Critical text meets WCAG AAA
   - Color is never sole information carrier

## 5. GLOW EFFECTS

### 5.1 Glow Principles

Glow effects are signature to M1SSION™:

**Glow Types:**
1. **Inner Glow** — Subtle luminosity within element
2. **Outer Glow** — Halo extending beyond element
3. **Drop Shadow Glow** — Colored shadow beneath element
4. **Text Glow** — Bloom effect around text

**Glow Rules:**
- Glow color matches element color
- Glow opacity typically 30-60%
- Glow blur radius proportional to element size
- Glow never obscures content

### 5.2 Standard Glow Definitions

**Subtle Glow**
```css
box-shadow: 0 0 10px rgba(0, 229, 255, 0.3);
```

**Medium Glow**
```css
box-shadow: 0 0 20px rgba(0, 229, 255, 0.4),
            0 0 40px rgba(0, 229, 255, 0.2);
```

**Intense Glow**
```css
box-shadow: 0 0 20px rgba(0, 229, 255, 0.5),
            0 0 40px rgba(0, 229, 255, 0.3),
            0 0 60px rgba(0, 229, 255, 0.2);
```

**Pulse Glow (Animated)**
```css
@keyframes pulseGlow {
  0%, 100% { 
    box-shadow: 0 0 20px rgba(0, 229, 255, 0.4); 
  }
  50% { 
    box-shadow: 0 0 30px rgba(0, 229, 255, 0.6); 
  }
}
```

### 5.3 Text Glow

**Heading Glow**
```css
text-shadow: 0 0 10px rgba(0, 229, 255, 0.5),
             0 0 20px rgba(0, 229, 255, 0.3);
```

**Emphasis Glow**
```css
text-shadow: 0 0 5px rgba(0, 229, 255, 0.4);
```

## 7. COLOR ACCESSIBILITY

### 7.1 Contrast Ratios

All color combinations are tested for accessibility:

| Combination | Ratio | WCAG Level |
|-------------|-------|------------|
| White on Void Black | 18.5:1 | AAA |
| Light on Void Black | 12.1:1 | AAA |
| Pulse Cyan on Void Black | 10.2:1 | AAA |
| Mist on Void Black | 5.8:1 | AA |
| Steel on Void Black | 4.1:1 | AA (large) |

### 7.2 Color Blindness Considerations

The palette is designed for color vision deficiency accessibility:

- **Protanopia/Deuteranopia:** Cyan and Magenta remain distinguishable
- **Tritanopia:** Sufficient luminance contrast maintained
- **All types:** Shape, position, and text support color meanings

### 7.3 Non-Color Indicators

Color is never the sole indicator of meaning:

- Icons accompany colored status indicators
- Text labels confirm color meaning
- Patterns or shapes provide additional distinction
- Animation reinforces state changes

## 8. COLOR APPLICATION RULES

### 8.1 Background Rules

- Primary backgrounds: Void Black (#0A0A14)
- Card/container backgrounds: Deep Space (#12121F)
- Interactive surfaces: Night Surface (#1A1A2E)
- Never use light backgrounds except in error states

### 8.2 Text Rules

- Primary text: Light (#C0C0E0)
- Secondary text: Mist (#606080)
- Disabled text: Steel (#454560)
- Headings may use accent colors with glow

### 8.3 Accent Rules

- Pulse Cyan for primary actions and positive states
- Signal Magenta for warnings and SHADOW content
- Echo Violet for special content and ECHO entity
- Success Green for confirmations and rewards
- Never mix multiple accents on same element

### 8.4 Entity Rules

- Entity color applied to associated content only
- Entity backgrounds at 10% opacity maximum
- Entity glows at 30-50% opacity
- Entity colors never on non-entity content
